## 📖 Schreibprinzipien

Jeder Text in diesem Repo folgt diesen Regeln:

1. **Datiert** — Inhalte veralten, das ist OK wenn transparent
2. **Begründet** — keine Behauptung ohne Quelle oder eigene Erfahrung
3. **Konkret** — abstrakte Aussagen sind verboten ("vielleicht", "tendenziell" raus)
4. **Kritisch** — auch eigene frühere Positionen werden hinterfragt
5. **Lesbar** — max. 3000 Wörter, max. 10 Min Lesedauer pro Artikel
6. **DSGVO-/Authentizitäts-konform** — keine Kundenfälle ohne Freigabe

## 🧩 Format eines Whitepapers

```markdown
# Titel

**Veröffentlicht:** YYYY-MM
**Lesedauer:** X Minuten
**Stand der Quellen:** YYYY-MM

## TL;DR
[3-5 Sätze, was der Leser mitnehmen soll]

## Worum es geht
[Kontext, warum das relevant ist]

## Hauptteil
[3-5 Abschnitte mit Zwischen-Überschriften]

## Was bedeutet das konkret?
[Handlungsempfehlungen für die Zielgruppe]

## Vorbehalte / Limitations
[Was diese Analyse NICHT macht — wichtig für Glaubwürdigkeit]

## Quellen
[Verlinkte Quellen mit Datum]

## Über den Autor
[2-3 Sätze, am Ende, nicht im Vordergrund]
```

## 🎓 Lessons Learned (aus der Praxis des Schreibens)

1. **Datum auf den Text — Pflicht.** Nichts veraltet schneller als
   IT-Empfehlungen.

2. **TL;DR ist nicht optional.** 80% der Leser lesen nur die ersten 3 Sätze.

3. **Vorbehalte machen den Text stärker.** Wer transparent über Grenzen
   spricht, wird ernst genommen.
